# 产品准则

## 语气与风格

- **专业与趣味并重：** 在文档和设置页面保持清晰、专业的语气，建立用户对数据同步和隐私安全的信任。在表情包选择器和交互提示中，采用更轻松、有趣的语调，呼应产品的核心用途。
- **清晰简洁：** 避免过多的技术术语，除非是在面向开发者的文档中。功能说明应直观易懂，让所有用户都能轻松上手。
- **鼓励探索：** 引导用户尝试 AI 命名、批量管理等高级功能，释放产品的全部潜力。

## 视觉识别

- **配色方案：**
  - **主色调：** 采用鲜明、现代的色彩（如 Ant Design Vue 的默认蓝色或 Tailwind 的 Primary 色系），传达活力与科技感。
  - **中性色：** 使用简洁的黑白灰作为背景和文字颜色，确保内容的可读性。
  - **深色模式：** 全面支持深色模式，使用深灰背景和柔和的文字颜色，减少视疲劳。
- **排版：** 使用现代无衬线字体（如 Inter, Roboto, 或系统默认无衬线字体），确保在各种屏幕尺寸和分辨率下的清晰度。
- **图标：** 使用 Ant Design 的图标库，保持风格统一、简洁。

## 用户体验 (UX) 原则

- **性能优先：** 表情包选择器必须秒开（<100ms）。任何可能导致卡顿的操作（如同步、批量处理）都应在后台进行，并提供非阻塞的进度提示。
- **无缝集成：** 扩展应尽可能“隐形”，只在用户需要时出现。注入的图标或快捷键不应干扰网页的正常功能。
- **即时反馈：** 用户的所有操作（添加、删除、重命名）都应有即时的视觉反馈。
- **隐私至上：** 明确告知用户数据的存储位置和同步方式，尊重用户的隐私选择。

## 开发准则

- **组件化：** 遵循 Vue 3 的组件化开发模式，保持代码的模块化和可复用性。
- **类型安全：** 严格遵守 TypeScript 规范，确保代码的健壮性和可维护性。
- **多层存储策略：** 始终遵循 Local -> Session -> Extension -> IndexedDB 的读写顺序，确保数据的一致性和高性能。
- **AI 使用规范：** 明确标识由 AI 生成的内容（如自动命名），并允许用户轻松修改。

## 品牌信息

- **核心口号：** "你的表情包，全网通用。"
- **使命：** 打破平台壁垒，让个性化表达无处不在。
